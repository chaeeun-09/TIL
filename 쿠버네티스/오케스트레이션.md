# 오케스트레이션이란?


분산된 환경에서 컨테이너의 수명 주기를 관리하는 프로세스이다.

> 수명주기란? <br>
컨테이너가(생성 → 실행 → 모니터링/관리 → 업데이트 → 종료)되는 전체 과정을 말한다.

이런 과정들을 사람이 하는게 아니라 프로세스가 자동적으로 관리 해주는 프로세스를 말한다.                        자동적인 과정을 통해 리소스를 효율적으로 사용, 안정성 향상 및 원활한 업데이트를 지원한다.

# 오케스트레이션 작동 방식


오케스트레이션은 여러 호스트 시스템 또는 클러스터 간의 컨테이너 배포를 조정하여 작동한다.                     오케스트레이션 플랫폼은 일련의 규칙 및 정책을 사용하여 컨테이너 수명을 관리한다.

> 클러스터 : 여러 호스트를 논리적으로 묶어서 하나의 시스템처럼 관리하는 그룹

- 스케줄링 : 미리 정해둔 조건, 우선 순위에 따라 사용가능한 호스트 리소스에 컨테이너 할당
- 서비스 검색  :  컨테이너 식별하고 연결하여 프로그램을 형성
- 로드 밸런싱 : 네트워크 트래픽을 컨테이너 간에 균등하게 분산하여 리소스 활용을 최적화하여 응용 프로그램 성능을 향상 시킨다.
- 스케일링 : 애플리케이션 수요에 따라서 컨테이너 인스턴스 수를 자동으로 조정
- 상태 모니터링 : 컨테이너 성능을 모니터링하고 장애가 발생한 컨테이너를 새 컨테이너로 교체
- 구성 관리 :  응용프로그램의 모든 컨테이너에서 일관된 구성을 보장
- 네트워킹 : 컨테이너와 외부 컨테이너 간 통신을 관리
